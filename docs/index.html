Overview


Aplikasi ini bernama iamra course, iamra merupakan nama saya sendiri yaitu Ilham Rahmat Akbar. Awalnya aplikasi ini dibuat untuk dokumentasi materi programming yang telah saya pelajari setelah saya lulus kuliah tahun 2022. Kemudian seiring waktu, materi tersebut cukup banyak dan ketika ingin dibuka lagi melihat kodingan lama, cukup berantakan. Maka dari itu supaya rapi saya buat folder dan disusun ulang berdasarkan bahasa programming. Sebenarnya sudah cukup rapi, tapi ketika di perjalanan semisal di krl atau di transjakarta ingin melihat materi kodingan, harapannya dengan adanya aplikasi ini bisa melihat materi programming secara online.

MVVM


Aplikasi ini dikembangkan menggunakan Clean Architecture yang dikombinasikan dengan pola desain MVVM (Model-View-ViewModel). Struktur direktori disusun secara modular untuk memisahkan business logic, UI logic, dan data handling. Pendekatan ini dipilih untuk memastikan kode yang scalable, mudah diuji (testable), dan memfasilitasi kolaborasi pengembangan fitur secara independen.

Layer ini merupakan lapisan terluar yang berinteraksi langsung dengan pengguna. Pengorganisasian paket di dalamnya menggunakan pendekatan Package by Feature, di mana setiap fitur memiliki direktori sendiri yang mencakup seluruh kebutuhan UI-nya.


Fragment/Activity
: Bertanggung jawab hanya untuk merender tampilan dan menangkap input pengguna (contoh: `CoursesFragment.java`, `EarningsFragment.java`).
ViewModel
: Menangani logika presentasi dan menyimpan state UI menggunakan `LiveData`/`StateFlow`. ViewModel berkomunikasi dengan Domain Layer untuk mendapatkan data (contoh: `CoursesViewModel.java`, `LoginViewModel.java`).
Adapter
: Menghubungkan data ke komponen list seperti RecyclerView (contoh: `CoursesAdapter.java`, `GroupAdapter.java`).

Ini adalah inti dari aplikasi yang berisi aturan bisnis murni dan tidak memiliki ketergantungan pada framework Android maupun library pihak ketiga. Layer ini mendefinisikan "apa" yang bisa dilakukan aplikasi.


/model/
: Mendefinisikan entitas data atau POJO (Plain Old Java Object) yang digunakan di seluruh aplikasi (contoh: `Course.java`, `User.java`, `Certificate.java`).
/repository/
: Berisi Interface yang berfungsi sebagai kontrak data. Interface ini menjelaskan data apa yang dibutuhkan oleh domain tanpa peduli dari mana asalnya (contoh: `StudentCoursesRepository.java`, `AuthRepository.java`).
/usecase/
: (Opsional) Menangani logika bisnis spesifik yang kompleks (contoh: `LogoutUseCase.java`).

Layer ini bertugas mengelola sumber data dan merupakan implementasi konkret dari kontrak yang ada di Domain Layer. Layer ini menerapkan pola Repository Pattern.


remote/ - Komunikasi Jaringan

/api/
: Definisi endpoint Retrofit (contoh: `StudentApi.java`, `InstructorApi.java`).
/dto/
: Data Transfer Objects untuk memetakan respons JSON dari server (Request & Response bodies).

local/ - Penyimpanan Lokal

/entity/
: Representasi tabel database (contoh: `StudentCoursesEntity.java`).
/dao/
: Data Access Objects yang berisi query SQL (contoh: `StudentCoursesDao.java`).

/mapper
: Bertugas mentransformasi data dari format DTO (Network) atau Entity (Database) menjadi format Model (Domain) agar lapisan domain tetap bersih (contoh: `CourseMapper.java`).
/repository
: Implementasi logika Single Source of Truth. Kelas ini (contoh: `StudentCoursesRepositoryImpl.java`) memutuskan apakah data diambil dari cache lokal atau API.

Direktori ini mengelola pembuatan dan penyuntikan dependensi menggunakan Dagger Hilt. Modul-modul dipisahkan berdasarkan fungsinya untuk memudahkan konfigurasi:


NetworkModule.java
: Menyediakan dependensi jaringan (Retrofit, OkHttp).
DatabaseModule.java
: Menyediakan dependensi database (Room).
RepositoryModule.java
: Menghubungkan Interface Repository di Domain dengan implementasinya di Data layer.

Modul pendukung yang berisi kode dasar dan fungsi bantuan yang digunakan secara global.
